# 掃除当番自動割り当てシステム 要件定義書

## 1. 開発背景・目的
「誰が何回やったか」を手動で管理する手間をなくしたいと考え、当番履歴をDBで管理して自動で公平に割り当てる仕組みを作る。PHP・MySQLの基礎をひととおり自力で実装することを目標とする。

---

## 2. 機能要件

### 当番表示機能（index.php）
- 今日の日付を自動取得して表示する
- 今日の当番をDBから取得して表示する
- 当番データがない場合はメッセージを表示する
- 当番割り当てページと管理画面へのリンクを表示する

### 当番割り当て機能（assign.php）
- 今日すでに割り当て済みの場合は処理を中断する（二重割り当て防止）
- 掃除場所の一覧をDBから取得する
- 社員を当番回数が少ない順に取得する
- 場所ごとに社員を割り当ててDBに保存する
- 処理完了後、当番表に戻る

### 管理機能（admin.php）
- パスワード入力で管理画面に入れる
- 社員の追加ができる
- 社員の論理削除（退職）ができる
- 退職済み社員の完全削除ができる
- 完全削除時は確認ダイアログを表示する
- 完全削除時はscheduleテーブルの関連データも削除する
- 掃除場所の追加ができる
- 掃除場所の削除ができる
- 履歴に残っている場所は削除できず、理由を表示する

---

## 3. 非機能要件

### セキュリティ
- SQLインジェクション対策：prepare/executeを使用
- XSS対策：htmlspecialcharsを使用
- 管理画面はパスワードで保護する

### データ整合性
- 外部キー制約でemployee_idとlocation_idの整合性を保つ
- 論理削除で退職社員の当番履歴を保持する

---

## 4. DB設計

### employeesテーブル
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INT | 主キー・自動採番 |
| name | VARCHAR(50) | 社員名 |
| is_active | TINYINT | 在籍フラグ（1:在籍 / 0:退職） |

### locationsテーブル
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INT | 主キー・自動採番 |
| name | VARCHAR(50) | 場所名 |

### scheduleテーブル
| カラム名 | 型 | 説明 |
|---|---|---|
| id | INT | 主キー・自動採番 |
| employee_id | INT | 社員ID（外部キー） |
| location_id | INT | 場所ID（外部キー） |
| date | DATE | 当番日付 |

### テーブルの関連
- schedule.employee_id → employees.id（外部キー）
- schedule.location_id → locations.id（外部キー）
